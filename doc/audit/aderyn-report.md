## Maintainer Notes (2026-03-02)

The items below annotate Aderyn’s findings with project context and any adjustments made since the report was generated.

- **L-2 Unspecific Solidity Pragma**: Intentionally left as `^0.8.27` to remain compatible with OpenZeppelin Confidential Contracts. Hardhat compiles with `0.8.34` (see `hardhat.config.ts`). This is a deliberate choice.
- **L-3 PUSH0 Opcode**: The build targets `evmVersion: "prague"` in `hardhat.config.ts`, which supports PUSH0. If you deploy to a chain without PUSH0 support, change the EVM target accordingly.
- **L-4 Modifier Invoked Only Once**: `onlySupplyPublisher` is kept for readability and consistency with the module pattern. No change required.
- **L-5 Empty Block**: Empty overrides are part of the module pattern and intentionally retain empty bodies for role gates and hooks.
- **L-6 Internal Function Used Only Once**: `initialize` is internal and called by the constructor to preserve the initializer pattern; keeping it is intentional.
- **L-7 Unchecked Return**: `FHE.allow` returns the same handle that was passed in (side-effect call), and `FHE.makePubliclyDecryptable` is also used for its side effects. We intentionally ignore these return values.

Additional manual adjustments made after the original report:
- Split supply roles: `SUPPLY_OBSERVER_ROLE` now gates total supply observer list management, and `SUPPLY_PUBLISHER_ROLE` gates `publishTotalSupply`.
- Added zero-address guard for `addTotalSupplyObserver` and a test for the edge case.
